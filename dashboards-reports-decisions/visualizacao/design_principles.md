# Princípios de Design e Acessibilidade em BI

Criar visualizações eficazes exige mais do que apenas habilidade técnica em ferramentas; exige uma mentalidade de design centrada no usuário.

---

## Metodologia: Design Thinking em BI

O design thinking divide o processo de criação em cinco estágios iterativos:
1. **Empatia**: Entender as necessidades e frustrações do stakeholder.
2. **Definição**: Clarificar a pergunta de negócio que o painel deve responder.
3. **Ideação**: Explorar múltiplas formas de organizar e visualizar os dados.
4. **Protótipo**: Criar versões de baixa fidelidade (mockups) para teste inicial.
5. **Teste**: Coletar feedback e refinar a solução.

---

## Princípios Fundamentais de Design

Aplique estes nove princípios para garantir que suas visualizações sejam equilibradas e comunicativas:

*   **Equilíbrio**: Distribuição uniforme dos elementos visuais.
*   **Ênfase**: Destacar o que é mais importante (através de cor ou tamanho).
*   **Movimento**: Guiar o olhar do usuário através do dashboard.
*   **Padrão e Repetição**: Criar consistência visual.
*   **Proporção**: Relação de tamanho entre os elementos baseado na relevância.
*   **Ritmo e Variedade**: Evitar a monotonia visual sem perder a coesão.
*   **Unidade**: Todos os elementos devem parecer parte de um mesmo sistema.

---

## Acessibilidade: Design para Todos

A acessibilidade não é opcional; é uma prática recomendada de design. Como profissional de BI, considere:

- **Daltonismo**: Utilize paletas de cores acessíveis e evite depender exclusivamente da cor para transmitir significado.
- **Rótulos e Texto Alternativo**: Use etiquetas claras para que a visualização possa ser compreendida sem esforço ou através de tecnologias assistivas.
- **Simplificação**: Remova o "lixo gráfico" (junk charts). Se um elemento não adiciona valor ao insight, ele deve ser removido.
- **Atributos Pré-atencionais**: Utilize marcas (pontos, linhas) e canais (cor, posição) para que as pessoas reconheçam informações automaticamente, sem esforço consciente.

## Frameworks de Referência

Para orientar seu processo, consulte estas estruturas consagradas:
- **Método McCandless**: Organização de pensamento sobre a beleza e utilidade da informação.
- **Junk Charts Trifecta Checkup (Kaiser Fung)**: Uma lista de verificação para avaliar se um gráfico comunica a verdade de forma precisa.
- **Vocabulário Visual (Financial Times)**: Um guia essencial para escolher o tipo de gráfico baseado na relação dos dados (disponível em pôster ou GitHub).

---

## Lista de Verificação (Checklist) de Design

- [ ] A visualização responde diretamente à pergunta de negócio?
- [ ] O gráfico escolhido é o mais adequado para o tipo de dado (dimensão/medida)?
- [ ] Os elementos mais importantes estão enfatizados?
- [ ] O dashboard é acessível para usuários com daltonismo?
- [ ] Foi removido o excesso de decorações desnecessárias?
